For Part Three, the option for the Buyer to walk away (purchase zero widgets) has been restored.

In Part Three of the experiment, you will make decisions as the Seller, choosing {{ if single == 1 }} an amount of widgets {{ else }} a menu of widgets {{ endif }} to offer to the Buyer. Recall that in Part One, your payoff as the Seller depended on two things you couldn't control: the ideal quantity drawn from the urn, and the specific Buyer you were matched with. In contrast, in Part Three we will remove this randomness, paying you a certain amount for any particular choice you might make. This certain amount of money on offer is the average expected payment from your choice over the {{ if single == 1 }} offer. {{ else }} menu. {{ endif }}

To generate this payment amount, instead of being matched to another participant in this room, you will be matched to a large number of Robot Buyers. Your payment will be the average Seller payoff across all of those matches. Each Robot Buyer has been trained using choices across made by {{ if single == 1 }}66{{ else }}64{{ endif }} real Buyer participants from earlier sessions of this experiment, who faced an identical Part One. Because of this training, for any {{ if single == 1 }} offer {{ else }} menu {{ endif }} you select, the Robot Buyers' decisions represent our best guess about how each of those {{ if single == 1 }}66{{ else }}64{{ endif }} prior participants would respond, across every ideal quantity that could be drawn from the urn.

For any {{ if single == 1 }} offer {{ else }} menu {{ endif }} you select in the Part Three decisions, you can think of your payment as being calculated by averaging the Seller's payoff across millions of different Robot Buyers responding to your offer. Each separate Robot Buyer represents a fair draw over both an ideal quantity from the given urn and of the likely choices from one of the {{ if single == 1 }}66{{ else }}64{{ endif }} prior participants.

Because the payment rule averages across millions of Robot Buyers, there is no longer any randomness in the payment for any specific {{ if single == 1 }} offer {{ else }} menu {{ endif }} you select as the Seller. Any choice you make will lead to a fixed additional payment, based on the 20 different balls that could be drawn from the urn to determine the Buyer’s ideal quantity and our best guess on the average outcome for the Seller payment across the {{ if single == 1 }}66{{ else }}64{{ endif }} prior participants responding to your offer.

Given this rule, your role in Part Three is to choose whichever {{ if single == 1 }} offer {{ else }} menu {{ endif }} you think will generate the highest average payment for the Seller.
